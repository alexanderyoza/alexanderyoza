# Project presets

Worked examples of how this skill applies to common app types. Each
preset is a *demonstration* of what the style router output looks like
for a concrete brief, not a template to copy verbatim.

Use a preset when:

- You want to learn how the router reasons.
- Your product is similar enough that the preset's diagnosis is a
  reasonable starting point (then customize from there).
- You want to see what a complete handoff to downstream prompts looks
  like.

Do **not** use a preset to skip the style router. The router exists so
each product gets its own answer.

## Available presets

- `sample-language-learning-app.md`: calm + AI-native productivity.
- `sample-photo-sharing-app.md`: social-mobile + playful-consumer.
- `sample-ai-notes-app.md`: AI-native productivity + premium editorial
  (or technical-devtool, depending on audience).
- `sample-finance-dashboard.md`: finance / trading + technical devtool.
- `sample-agency-website.md`: premium editorial + creative studio
  portfolio (or luxury commerce for high-end clientele).

## Format of each preset

Every preset includes:

1. **Style diagnosis**: the answers to the router's 10 questions.
2. **Primary style**: chosen reference + why.
3. **Secondary style**: chosen reference + what it contributes.
4. **Visual rules**: 5–10 rules derived from the chosen pair.
5. **Anti-patterns**: what this product must avoid.
6. **Suggested components**: core component patterns to use.
7. **Motion / interaction direction**: how things should move.
8. **Sample design-system notes**: token-level starting points.
9. **Sample page override**: one screen's worth of override rules.
10. **Suggested `/uiux` workflow**: which commands to run, in order.
