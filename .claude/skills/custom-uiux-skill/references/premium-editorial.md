# Reference direction: Premium Editorial

Long-read magazines, thoughtful publications, premium content products,
and websites that take typography seriously. The visual language borrows
from print art direction more than from product design.

## When to use it

- Reading-heavy products (articles, essays, newsletters, books).
- Premium media, journalism, research, or curated content brands.
- Thought-leader or knowledge-business landing pages.
- Marketing surfaces for products that want to feel *considered*.

## When not to use it

- High-density data tools: editorial reads as wasteful at scale.
- Mobile-only utilities where you cannot afford big type.
- Playful consumer products: the gravity reads as joyless.
- Workflows requiring many tightly packed actions per screen.

## What it should feel like

Considered. Quiet. Confident. The reader senses an editor was here. Pages
breathe. The typography carries the brand. Color and image are
supporting cast.

## Layout traits

- Tall, generous vertical rhythm; baseline-aware spacing.
- 1–2 dominant columns; the body is narrow (60–75ch).
- Strong asymmetry permitted: lead images bleed, pull quotes interrupt.
- Hero is often a single large headline + small dek + a date or byline,
  not a SaaS hero with CTAs everywhere.
- Wide margins on desktop. Mobile drops to a single 16–20px-padded column.

## Color tendencies

- Mostly monochrome (off-white / paper, ink black, one warm accent).
- One signature accent, used sparingly (rules, links, pull quotes).
- Avoid candy color, neon, and gradient washes.
- Light mode default; if dark, it should feel like a *night edition*,
  not a tech product.

## Typography tendencies

- Serif display (Tiempos, GT Sectra, Domaine, Source Serif, Caslon
  family, etc.) for headlines and pull quotes.
- Highly legible humanist sans for UI and metadata.
- Two-face system is the norm. Three faces only with deliberate intent.
- Italics treated as a real expressive tool. Drop caps welcomed on
  long-reads.
- Body type 18–20px on desktop, 16–17px on mobile, line-height 1.5–1.7.

## Component patterns

- Article-card grid with a clear "lead story" treatment.
- Byline rows with thin rules above/below.
- Sidenotes and margin notes (esp. on wide layouts).
- Quiet buttons (text + hairline border, or all-text + underline).
- Footnote-style links and citation patterns.
- Tags as text, not as chips with heavy backgrounds.

## Navigation patterns

- Horizontal top nav with masthead-style logotype.
- Sections labeled like magazine sections (Issues, Long Reads, Index).
- Breadcrumbs read as "Volume / Issue / Section."
- On mobile, a single-column drawer with strong typography, not a
  cluttered grid of icons.

## Motion / interaction style

- Subtle. Things appear, they don't bounce.
- 150–250ms ease-out fades, slight upward translate on enter.
- No parallax tricks beyond the lead image. No spring physics.
- Reading-position indicators are welcome; scroll-jacking is not.

## Good inspiration categories

- Independent print magazines (Apartamento, Kinfolk, The Gentlewoman).
- Long-read web (The New Yorker article pages, Aeon, The Atavist).
- Type foundry websites.
- Museum catalog sites.

(Look at categories, not specific brands. Do not copy.)

## Anti-patterns to avoid

- Sparkle/star icons or AI-gradient hero blobs.
- Centered hero with three CTAs.
- Bento grid feature blocks.
- All-`rounded-2xl` everything.
- Stock photos of teams pointing at laptops.
- Animated number counters in the hero.
- Tag clouds with bright pill backgrounds.

## Example suitable products

- A research org's publication site.
- A newsletter platform's reader.
- A book publisher's web presence.
- A premium podcast network landing site.
- A think-tank's report library.

## Example unsuitable products

- A live trading dashboard.
- A teen-oriented social app.
- A consumer fitness tracker.
- A POS system used by warehouse staff.
- A real-time multiplayer game lobby.
