# Reference direction: Creative Studio / Portfolio

Boutique design studios, freelancers, photographers, illustrators,
architects, agencies. The site itself is a portfolio piece. The product
is the *taste*. Less "showcase grid of thumbnails," more "this person
clearly knows what they're doing."

## When to use it

- Independent designer / photographer / illustrator portfolios.
- Boutique design or branding agency sites.
- Personal brand sites for senior creative professionals.
- Studio sites where the goal is inquiries from sympathetic clients,
  not volume.

## When not to use it

- Job-board / freelance-marketplace listings (use commerce instead).
- Mass-market personal sites where ease beats taste.
- The product surfaces of any of the above (tools, dashboards).
- Anything where the user is "doing work" inside the site.

## What it should feel like

Considered. Distinct. Each project the user clicks into feels staged.
The home page is not a wall of thumbnails. It's a moment.

## Layout traits

- Asymmetric, generous. Project tiles can be enormous.
- Mixed media on the home: a giant image, a quote, a snippet of
  process, a single project title set in display type.
- Project detail pages are immersive: full-bleed media, considered
  type rhythm, almost no UI chrome.
- Mobile collapses to a single-column scroll narrative; not a 2-up
  thumbnail grid.

## Color tendencies

- Often one of: pure off-white + ink, deep-warm-black + cream, or a
  signature studio color used as the entire backdrop.
- Project pages may invert the studio palette to feature client work.
- No gradients (unless they're an editorial reference to a project).
- Color is decision-making, not decoration.

## Typography tendencies

- A confident display face (often a custom or quirky serif/sans).
- A workhorse second face for body and metadata (Inter, Söhne, ABC
  Diatype, etc., pattern-level, not specific assets).
- Large display sizes: 80–160px on desktop hero.
- Generous tracking and leading; never default Tailwind defaults.

## Component patterns

- Project tiles with a single thumbnail and a confidently-set title.
- Hover treatments: image swap, color shift, a small reveal of a
  client name or year.
- "About" page that reads like a short essay, not a SaaS team page.
- Contact module that is a sentence, not a form: "say hi at
  hello@studio.xx".
- Case-study layouts: hero image, problem, approach, outcome, all
  set in editorial type.

## Navigation patterns

- Few items: Work / Studio / Journal / Contact (or fewer).
- Logotype that is itself a designed mark, not Tailwind text.
- Footer: address, email, year, social (short and dignified).

## Motion / interaction style

- Restrained but expressive, a single signature move per page
  (a reveal, a marquee, a slow horizontal scroll).
- Project intros: image scales up from a small thumb to full bleed.
- Custom cursor on desktop only if the studio's brand earns it.
- Reduced motion respected; do not require motion to read the content.

## Good inspiration categories

- Independent design studios' own websites.
- Photographers' portfolios.
- Architects' practice sites.
- Illustrators' personal sites.
- Award-galleries that link to studios (use as inspiration *category*,
  not as templates to copy).

## Anti-patterns to avoid

- A 4×N grid of identical-size project thumbnails. That's a Dribbble
  feed, not a portfolio.
- Generic "Hi, I'm [name], a designer who loves clean design" headlines.
- Stock testimonials.
- "Trusted by" logo strip stuffed with client logos at the top.
- Reading the resume in long bullet lists on the home page.
- A blog tab that has one post from 2019.

## Example suitable products

- A solo designer's portfolio.
- A boutique branding agency's site.
- A photographer's archive.
- A small architecture practice's site.
- A typographer / type foundry's site.

## Example unsuitable products

- A devtool.
- A trading dashboard.
- A consumer social app.
- A wellness app.
- A children's product.
